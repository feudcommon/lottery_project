# SCAI Lucky Loop — Contracts

Solidity smart contract for **LuckyLoopToken (LLT)** — the ERC-20 reward token that SCAI Lucky Loop coins convert into on withdrawal. Built and tested with Hardhat 3 (`mocha` + `ethers`).

## Contract overview — `contracts/LLT.sol`

`LuckyLoopToken` is a standard ERC-20 (via OpenZeppelin 4.9.2) with three extensions:

- **`Ownable`** — mint and batch-transfer are owner-only. The backend's payout wallet is the owner.
- **`ERC20Burnable`** — anyone can burn their own balance (`burn`), or burn from an approved allowance (`burnFrom`).
- **`batchTransfer(recipients[], amounts[])`** — owner-only helper to pay out multiple addresses in a single transaction; reverts if the arrays are mismatched in length or the owner's balance is insufficient.

Key parameters:
- Name / symbol: `Lucky Loop Token` / `LLT`
- Decimals: `18` (hardcoded override, ignores ERC-20 default)
- Initial supply: `1,000,000 LLT`, minted to the deployer on construction

**Deployed instance (SCAI Private Network):**

- Chain ID: `34`
- RPC: `https://mainnet-rpc.scai.network`

## Setup

```bash
npm install
```

> **Note:** contract imports use Remix-style versioned paths (`@openzeppelin/contracts@4.9.2/...`), which Hardhat's `node_modules` resolution doesn't understand natively. If you hit `HHE902: The package "@openzeppelin/contracts@4.9.2" is not installed`, create a junction/symlink so the versioned folder name resolves:
>
> ```powershell
> # Windows
> New-Item -ItemType Junction -Path "node_modules\@openzeppelin\contracts@4.9.2" -Target "node_modules\@openzeppelin\contracts"
> ```
> ```bash
> # macOS/Linux
> ln -s node_modules/@openzeppelin/contracts "node_modules/@openzeppelin/contracts@4.9.2"
> ```
> See `LLTTestingLog.md` for the full story on why this exists rather than editing the import paths directly.

```bash
npx hardhat test solidity

```

`test/LLT.ts` covers 13 cases across deployment, minting, batch transfers, and burning — see `LLTTestingLog.md` for the full test log, every issue hit getting the suite passing (OZ v4 vs v5 revert-string differences, import resolution, etc.), and a follow-up log of live withdrawal-flow debugging against the deployed contract.

## Deploying

Local/simulated chain:

```bash
npx hardhat ignition deploy ignition/modules/Counter.ts
```

To Sepolia (or another real network), set a private key via the `hardhat-keystore` plugin:

```bash
npx hardhat keystore set SEPOLIA_PRIVATE_KEY
npx hardhat ignition deploy --network sepolia ignition/modules/Counter.ts
```

`hardhat.config.ts` also defines `hardhatMainnet` / `hardhatOp` simulated networks for local testing without touching a real chain. To deploy against the SCAI Private Network specifically, add a matching network entry (RPC URL + chain ID above) and a funded deployer key, then point ignition/`scripts/deploy.ts` at it.

## Solidity version note

Contracts pin `pragma solidity ^0.8.19;`, but the configured compiler is `0.8.28` (`hardhat.config.ts` → `solidity.profiles.default.version`). The caret allows any `0.8.x >= 0.8.19`, so this is expected — not a mismatch. To pin exactly to `0.8.19`, change that config value.

## Generated types

`types/ethers-contracts/` holds Typechain-generated TypeScript bindings (`LuckyLoopToken.ts`, `LuckyLoopToken__factory.ts`) for typed contract calls from Node/backend scripts — **autogenerated, do not edit by hand.**
